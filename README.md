MLOps Pipe Defect Detection Bot (YOLOv8 + Celery + FastAPI)

Telegram-бот для автоматизированной дефектоскопии труб. Бот принимает фотографии и видео с камер внутритрубных роботов, находит дефекты/трещины с помощью нейросети и поддерживает цикл непрерывного обучения (Continual Learning) на основе обратной связи от оператора.

### Используемые технологии:
* **CV Model:** YOLOv8s (Ultralytics) + OpenCV
* **Backend:** FastAPI
* **Task Broker:** Celery + Redis
* **Telegram Bot:** aiogram 3
* **Infrastructure:** Docker & docker-compose

### Основной функционал (Бот умеет обрабатывать):

📷 **Пакеты фотографий (Batch Processing)**
* Обработка альбомов фото (до 100 кадров) без зависаний сервера.
* **Silent Drop (Умный фильтр):** бот присылает в чат *только* те фотографии, на которых был обнаружен дефект, отсекая пустые кадры трубы для экономии времени оператора.

🎥 **Видеоинспекцию (Smart Frame Extraction)**
* Прием видеофайлов и их асинхронная обработка.
* Оптимизированная нарезка потока (2 кадра в секунду) для снижения нагрузки на GPU.
* Точный расчет таймкодов обнаруженных дефектов (вывод в формате: `Дефект на 01:24`).

🔄 **Непрерывное обучение (Continual Learning / Human-in-the-Loop)**
* Под каждым найденным дефектом доступны Inline-кнопки: `✅ Верно`, `⚠️ Неточная рамка`, `❌ Ложное срабатывание`.
* Автоматический сбор датасета:
  * **True Positives:** переносятся в папку `new_data` с размеченным BBox.
  * **False Positives:** исходный кадр переносится с пустым файлом разметки (Background Image), чтобы нейросеть училась игнорировать этот паттерн в будущем.

### ⚙️ Архитектура и отказоустойчивость (System Design)

Система спроектирована с учетом высоких нагрузок и защиты от исчерпания ресурсов:
* **Асинхронные очереди (Resource Isolation):** Логическое разделение задач внутри брокера сообщений. Отдельная очередь для фото (`celery`) и отдельная для видео (`video_queue`). 
* **Load Shedding:** Умный сброс нагрузки (Fail-Fast) при превышении жестких лимитов (`PHOTO_HARD_LIMIT = 100`, `VIDEO_HARD_LIMIT = 10`), предотвращающий Starvation (голодание) очередей.
* **Fail-Fast (Быстрый отказ):** Валидация лимитов очередей и размера файлов (защита от дискового переполнения) происходит на стороне FastAPI до физического сохранения тяжелых видеофайлов на жесткий диск сервера. Это исключает лишнюю нагрузку на Disk I/O и мгновенно освобождает оперативную память.
